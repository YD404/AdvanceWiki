ADVゲーム シナリオ構文仕様書

Googleドキュメント（またはテキストエディタ）で執筆されたシナリオをUnityで読み込み、実行するための記述ルール定義。

1. 基本ルール

ファイル形式と保存

  • 文字コード: UTF-8
  • 改行コード: LF または CRLF（Unity側で自動統一）
  • 保存場所: Assets/Resources/Scenario

読み込み順序

ファイル名順に結合されて1つのシナリオとして扱われます。

推奨ファイル名 01_prologue.txt, 02_chapter1.txt ...
特記事項

各テキストファイルの 1行目は「ファイル名（識別用）」 として扱われ、ゲーム内では無視されます。
シナリオ本文は2行目から記述してください。

2. テキスト表示

会話（セリフ）

カギカッコ 「 または 『 を含む行は、自動的にセリフとして処理されます。

名前あり

太郎「こんにちは」

名前省略（話者継続）

「昨日はよく眠れた？」

ト書き（演出・動作）

丸カッコ （ または ( で始まる行。名前欄非表示、カッコの中身だけ表示。

（激しい音が鳴り響く）

地の文（ナレーション）

上記以外。名前欄非表示。

雨が降り出した。
辺りは静寂に包まれている。

3. 演出コマンド

行頭の記号で背景変更やサウンド再生（全角・半角可）。

背景変更

行頭 ○ or @

○教室
@夕暮れの公園

サウンド再生

行頭 ♪

♪BGM：日常のテーマ
♪BGM：停止
♪SE：爆発音

4. 分岐と進行制御

ラベル（ジャンプ先）

行頭 *。Googleドキュメント「見出し1」で自動付与。

*route_A

選択肢

★ 文言、直下に → 飛び先。

★右の道を行く
→*route_right

★左の道を行く
→*route_left

強制ジャンプ

goto *common_route

5. その他

コメントアウト

行頭 // で無視。

// ここから第2章のデータ

6. パラメータ操作

行頭 #。内部計算のみ。

演算子 機能
+ 加算 (#好感度 + 1)
- 減算 (#HP - 10)
= 代入 (#Flag = 1)

完全なシナリオファイルの例です。コピーしてテスト用に使用してください。
